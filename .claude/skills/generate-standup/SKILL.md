# Generate Standup Skill

Run all steps in order. Do not skip any step.

## Step 1 — Fetch the standup

```bash
curl -s http://localhost:8000/standup
```

If the request fails or returns non-JSON, stop and tell the user:
> "The DailyFlow backend isn't running. Start it with: `cd backend && uvicorn main:app --reload`"

The response is a JSON object with exactly one field:
- `standup` — a prose string generated by AI

If `standup` equals `"No tasks were completed today yet."`, stop and tell the user:
> "No tasks have been completed today. Complete at least one task in DailyFlow before generating a standup."

Otherwise store the value of `standup` as `{raw_standup}`. It is a paragraph of prose describing:
- What was completed (specific tasks and subtasks accomplished)
- What is coming up next (pending tasks or planned work)

## Step 2 — Ask about blockers

Ask the user:
> "Any blockers to add? (type them or press Enter to skip)"

Wait for the response. Store it as `{blockers}`. Treat Enter / "no" / "none" / "n" as empty.

## Step 3 — Format and display the standup

Parse `{raw_standup}` and split it into two parts:
- Completed work: sentences referencing past tense actions, finished tasks, or "today I..."
- Next up: sentences referencing upcoming work, plans, or "looking ahead..."

If the prose doesn't cleanly split, use the first half as completed and the second as next up.

Strip any markdown formatting (e.g. `**bold**`) from the text before displaying.

Display:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  DAILY STANDUP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ✅  WHAT I COMPLETED
  {completed work — use bullet points if multiple items}

  🔨  WHAT I'M WORKING ON NEXT
  {upcoming work — use bullet points if multiple items}

  🚧  BLOCKERS
  {blockers if non-empty, otherwise "None"}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Also build a `{clipboard_text}` plain-text version (no box-drawing characters, no emoji) for pasting into Slack or tickets:

```
Yesterday/Today:
{completed work as plain text}

Working on next:
{upcoming work as plain text}

Blockers:
{blockers or "None"}
```

## Step 4 — Offer clipboard copy

Ask:
> "Copy to clipboard? (y/n)"

Wait for response.

If **y** or **yes**:
1. Detect OS:
   ```bash
   uname -s
   ```
2. Copy using the right tool:
   - `Darwin` → `echo "{clipboard_text}" | pbcopy`
   - `Linux` → try `which xclip` first; if found use `echo "{clipboard_text}" | xclip -selection clipboard`; else try `which xsel`; if found use `echo "{clipboard_text}" | xsel --clipboard --input`; else print `{clipboard_text}` and say "No clipboard tool found — copy the text above manually."
   - `MINGW*` / `CYGWIN*` / Windows → `echo "{clipboard_text}" | clip`
3. Confirm: "Copied to clipboard."

If **n** or **no**: say "Standup is ready above whenever you need it."

## Notes

- Read-only. Never modify tasks or backend state.
- `standup` from the API is a single prose paragraph — split it by meaning, not by line breaks.
- The clipboard version intentionally uses plain text so it pastes cleanly into Slack, Teams, or Jira.
- Preserve the AI's wording where possible; only reformat structure, don't paraphrase.
